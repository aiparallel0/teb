Architecture diagram for teb v2: a context-window-first design with strict file size limits, one-responsibility modules, and compile-time error enforcement replacing tests

The core constraint — every architectural decision follows from this
Context window = hard limit. Design around it, not through it.
An agent touching a file must load: the file itself + its direct interface contracts + nothing else. If understanding a file requires reading 3 other files first, the architecture has already failed. Target: any single agent task fits in <8KB of context.
Architecture laws
1 file = 1 responsibility, hard cap 200 lines
If a file exceeds 200 lines it has more than one job. Split it. No exceptions. 100KB files are an AI agent death sentence.
Interfaces at boundaries, never inside modules
Every cross-module call goes through a typed interface file. The agent writing module A never needs to read module B — only B's interface header. This is the C header model applied universally.
Flat imports, no transitive dependencies
Module A imports from B. A never imports from C through B. Transitive dependency chains mean an agent must follow the chain to understand anything — that's recursive context explosion.
Error at compile, not at runtime, never at test
Tests find what the type system missed. If your type system is strong enough, tests find almost nothing. Invest in compiler strictness, not test count. 0 tests + strict compilation > 1000 tests + dynamic typing.
Explicit over implicit, always
No magic. No reflection-based dependency injection. No dynamic dispatch where static dispatch works. An agent must be able to read a file and know exactly what it does — not what it might do at runtime.
Language recommendation
Rust primary recommended
The compiler IS the test suite. Every function signature is a contract. Every Result<T,E> forces you to handle the error path — no silent failures. The module system enforces the architecture laws above at compile time: you cannot have circular dependencies, you cannot accidentally make something public, you cannot misuse a type across a boundary.

Flags for teb2:
#![deny(warnings)]
#![deny(unused)]
#![deny(dead_code)]
#![forbid(unsafe_code)] // unless explicit
// Every warning = build failure
// unused import = build failure
// dead function = build failure
Zig alternative
Comptime replaces macros + generics cleanly. No hidden control flow — what you read is what runs. Excellent for the execution engine layer. Pre-1.0 ecosystem risk.
Go agents layer only
Goroutines + channels are the ideal model for multi-agent message passing. Simpler than Rust for this specific problem. Could be the agent orchestration layer while Rust owns the core.
Proposed file structure — context-budget per directory
teb2/
├── core/ // <200 lines each. zero external deps
│ ├── types.rs // ALL domain types. one truth. agents read this first
│ ├── errors.rs // ALL error variants. exhaustive match enforced
│ └── config.rs // env/config. no logic, pure data
├── auth/ // owns: jwt, argon2, rbac. nothing else
│ ├── mod.rs // public interface only — what callers see
│ ├── hash.rs // argon2id. 1 function, 1 purpose
│ ├── token.rs // jwt sign/verify. algorithm pinned at compile time
│ └── rbac.rs // role checks. pure functions, no db
├── agents/ // each agent = 1 file. agent reads only its own file + core/types
│ ├── coordinator.rs
│ ├── finance.rs
│ ├── outreach.rs
│ └── channel.rs // typed message enum — all inter-agent comms
├── exec/ // http execution engine. no agent logic here
│ ├── mod.rs
│ ├── http.rs // reqwest wrapper. typed requests only
│ └── vault.rs // chacha20poly1305. encrypt/decrypt. that's it
├── db/ // sqlx. queries verified at compile time against schema
│ ├── mod.rs
│ ├── goals.rs // ONLY goal queries
│ ├── tasks.rs // ONLY task queries
│ └── schema.sql // single source of truth for sqlx compile check
└── api/ // axum routes. thin layer. no business logic
├── mod.rs
├── goals.rs // goal routes only
└── tasks.rs // task routes only
