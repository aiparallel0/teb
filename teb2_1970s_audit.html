C itself was written for PDP-11. Unix ran in 64 KB. This is the target.
architecture layer audit
File structure + module boundariespasses
200-line cap, 2-in/1-out, flat includes, no transitive deps — this is exactly how C was written on PDP-11. K&R C was designed around this constraint because the machines had no choice. A PDP-11 linker could handle this file structure without modification. The header/.c split maps directly to how early Unix libraries were distributed on tape.
C99 language standardmostly passes — with a downgrade
C99 didn't exist in 1970 — K&R C (1978) and then ANSI C89 were the standards. However C99 is a strict superset and the features you're using (stdint.h, bool, uint64_t) are conservative additions. On a 16-bit PDP-11 you would replace int64_t with two uint32_t words and write explicit carry arithmetic. The architecture survives — the integer types need a hardware-aware typedef layer.
AgentMsg tagged union / channel.hpasses — this is 1970s idiom
Tagged unions (struct { enum tag; union payload; }) are pure K&R C. Message-passing via typed queues was the standard IPC model on early minicomputers before sockets existed. The channel design is closer to 1970s thinking than modern microservice design. A VAX engineer would recognise this immediately.
SQLite / db layerfails as-is — concept survives
SQLite as a library is a 2000s artifact — 140 KB of compiled code, WAL journaling, B-tree pages. A PDP-11 with 56 KB RAM cannot load SQLite. The concept — typed query structs in, typed row structs out — maps directly onto flat-file or ISAM databases of the era (VSAM on IBM, dbm on Unix). The interface contract survives; the implementation does not.
Argon2id / ChaCha20-Poly1305fails — era mismatch
Argon2 requires intentionally large memory (64 MB default) — a PDP-11 has 56 KB total. ChaCha20 was designed in 2008. Neither existed nor could run. 1970s equivalent: DES (1975, NSA), or for purely military machines, KL-7 cipher hardware. The vault.c interface (bytes + key → ciphertext) survives perfectly — swap the algorithm, keep the contract.
HTTP client / exec layerfails — TCP/IP didn't exist yet
TCP/IP was standardised in 1974 (ARPANET), but HTTP is 1991. A 1970s machine doing "HTTP requests" would be doing serial RS-232 or BISYNC messages to a mainframe. The HttpReq → HttpResult interface shape is sound — it's just called something different. A 1970s rewrite replaces it with a serial message frame struct. The module boundary is correct; the protocol is wrong.
AI agent orchestration (the whole point of teb)fundamentally impossible
A PDP-11 running at 1 MIPS with 56 KB RAM cannot run an LLM. GPT-4 inference requires ~700 GB of weight memory. Claude requires similar. The multi-agent coordinator, the AI decomposition engine, the coaching layer — none of these run locally on 1970s iron. What could run: the task queue, the rule-based nudge system, the financial pipeline arithmetic, the credential vault (with DES), the HTTP-equivalent serial comms. teb without AI is a structured task tracker — and that absolutely runs on a VAX.

Honest verdict: the skeleton is 1970s. the organs are not.
The architectural principles — flat modules, typed I/O contracts, message-passing agents, compile-time error enforcement, no transitive deps — are not modern ideas dressed up in retro clothing. They are genuinely how disciplined systems were built on PDP-11 and VAX iron, because the machines gave you no alternative. You rediscovered them from first principles by treating the AI context window as the hard constraint, which is the same pressure the 64 KB RAM limit applied in 1973.

What fails is entirely the implementation stack, not the design. SQLite, Argon2, ChaCha20, HTTP, and LLM inference are all post-1990 artifacts. Replace them with era-appropriate equivalents — flat file I/O, DES, serial framing, rule-based expert systems — and the architecture runs unchanged on a VAX-11/780.

The most honest statement: a 1970s engineer handed this architecture document would nod and start coding. a 1970s engineer handed the implementation dependencies would not recognise them.
