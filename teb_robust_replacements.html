Component-by-component replacement recommendations for the teb project, swapping Python for more robust language alternatives

Robustest replacement
Why + migration effort
Web API / server
Python FastAPI 0.115
Rust Axum + Tower
hard migration max robustness
Axum gives compile-time request/response types, zero-copy extractors, and Tower middleware. No runtime type errors. Memory-safe. Handles 10–50× more requests/sec. Migration is a full rewrite.

Pragmatic alternative: Go Gin or Chi — 80% of the safety gain, much faster migration, excellent stdlib HTTP.
Authentication / JWT
Python bcrypt + custom JWT + RBAC
Rust argon2 + jsonwebtoken
hard migration stronger crypto
Argon2id is the current gold-standard password hash (winner of Password Hashing Competition). The Rust jsonwebtoken crate enforces algorithm at compile time — no "alg: none" attack surface. bcrypt is fine but Argon2 is strictly better for new code.

Pragmatic alternative: Stay Python but swap bcrypt → argon2-cffi and use python-jose with explicit algorithm pinning.
Database / ORM
Python SQLite + custom SQL migrations
Rust SQLx + SQLite
hard migration compile-time SQL
SQLx verifies every SQL query against the real database schema at compile time — a query with a wrong column name fails the build, not at 3am. Zero runtime query surprises. Keeps SQLite, so no infra change.

Pragmatic alternative: Python swap raw SQL for SQLModel or SQLAlchemy 2.0 with strict typing — catches most ORM-level bugs before runtime.
Async HTTP client
Python httpx (async)
Rust reqwest
hard migration max robustness
reqwest is memory-safe, async (Tokio), has compile-time-typed request/response builders, and handles TLS via rustls (no OpenSSL footguns). Connection pool management is deterministic.

Pragmatic alternative: httpx is already excellent. Pin versions and add strict timeout config — this is a low-priority swap.
Browser automation
Python Playwright
Rust chromiumoxide
hard migration niche ecosystem
chromiumoxide drives Chrome via CDP in async Rust. Type-safe, memory-safe, very fast. But the ecosystem is immature vs Playwright.

Pragmatic alternative: Keep Playwright-Python — it's the most robust browser automation tool that exists in any language. This is the one component where Python wins outright. Playwright's reliability, cross-browser support, and test recording make it the right choice regardless of language preference.
Credential vault
Python Fernet encryption
Rust age + ring / chacha20poly1305
medium migration modern AEAD
Fernet is AES-128-CBC + HMAC-SHA256 — solid but not modern AEAD. ChaCha20-Poly1305 (via the ring crate) or age encryption give authenticated encryption with simpler, harder-to-misuse APIs and no padding oracle surface.

Pragmatic alternative: Stay Python, swap Fernet → cryptography lib's AESGCM or ChaCha20Poly1305. Same language, better primitive.
Multi-agent orchestration
Python custom message passing, 6 agents
Go goroutines + channels
medium migration race-free concurrency
Go's goroutines + typed channels make the message-passing architecture explicit and race-condition-free. The Go race detector (-race) catches concurrency bugs at test time. Python's GIL and asyncio interplay is a footgun for multi-agent work.

Pragmatic alternative: Rust's tokio::mpsc channels give the same guarantees with compile-time enforcement — but Go is the pragmatic win here for this use case.
Financial pipeline
Python Mercury + Stripe integrations
Rust async-stripe + custom Mercury client
hard migration safest for money
Money code has zero tolerance for type errors, silent integer overflow, or race conditions. Rust's type system can encode "approved amount" vs "pending amount" as distinct types — the compiler prevents mixing them. Idempotency keys enforced by type.

Pragmatic alternative: Stay Python but wrap all financial operations in strict Pydantic v2 models with Decimal (never float) and transaction-level DB locks.
Recommended migration path for maximum robustness
Phase 1 (stay Python, big wins): Argon2 for passwords · SQLModel/SQLAlchemy 2.0 · Pydantic v2 everywhere · Decimal for all money · ChaCha20-Poly1305 for vault · pin all deps with hashes
Phase 2 (Go for agents): Rewrite the multi-agent layer in Go — goroutines + channels are the best fit for your message-passing design · Keep Python/Playwright for browser tasks, call out via subprocess or gRPC
Phase 3 (Rust for critical path): Rewrite the financial pipeline and auth service in Rust · Axum API in front, Python Playwright workers behind · Full rewrite only if scale or security demands it
