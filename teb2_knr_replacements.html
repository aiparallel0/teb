DES — Data Encryption Standard
NIST-standardised 1977. Designed explicitly for hardware and early C. libdes shipped on BSD. 8-byte blocks, 56-bit key. Same vault.c interface: Bytes + Key → CipherResult. Swap to AES-128 on modern builds via typedef.
era-native
/* vault.c — K&R */

Bytes plain;
Key8 key;
{
  CipherResult r;
  des_ecb_encrypt(&plain, &r.ct, key, DES_ENCRYPT);
  r.err = ERR_OK;
  return r;
}

RS-232 serial frame / UUCP protocol
UUCP (Unix-to-Unix Copy) shipped 1978. Serial framed messages with checksum. Same http.c interface: Req + Cred → Result. On modern builds the same struct goes over TCP. The frame format is identical — only the transport changes.

Rule table + decision tree agent
MYCIN (1976) and DENDRAL (1969) were expert systems running on 1970s iron with exactly this pattern: AgentMsg → rule_table[] → AgentMsg. Each agent is a switch over message type, dispatching to a hand-coded rule. No inference — pure lookup. On modern builds, swap rule_table dispatch for an LLM call behind the same interface.
era-native
/* coord.c — K&R */
coord_handle(msg)
AgentMsg msg;
{
  switch (msg.tag) {
    case MSG_GOAL_NEW:
      return decompose_rule(msg);
    case MSG_TASK_DONE:
      return next_task_rule(msg);
    default:
      return msg_err(ERR_UNKNOWN);
  }
}

K&R C 1978 — full compliance changes
Every C99 feature used in teb2 — mapped to K&R equivalent
/* DROPPED from C99 → K&R */

/* 1. stdint.h types — use manual typedefs in types.h */
  C99: uint32_t, int64_t, bool
  K&R: typedef unsigned long u32; /* 32-bit on PDP-11 */
      typedef long i32;
      typedef long[2] i64; /* two-word 64-bit */
      #define TRUE 1
      #define FALSE 0

/* 2. Function prototypes — K&R has none */
  C99: int hash_password(const char *pw, HashConfig cfg);
  K&R: int hash_password(); /* declared in .h, defined in .c */

/* 3. K&R function definition style */
  C99: int hash_password(const char *pw, HashConfig cfg) { }
  K&R: int hash_password(pw, cfg)
      char *pw;
      HashConfig cfg;
      { }

/* 4. No // comments — C89+ only */
  K&R: /* always block comments */

/* 5. No const keyword — not in K&R */
  C99: const char *secret
  K&R: char *secret /* discipline only, no enforcement */

/* 6. No named initialisers */
  C99: HashResult r = { .hash = h, .err = ERR_OK };
  K&R: HashResult r; r.hash = h; r.err = ERR_OK;

/* 7. Compiler flags — replace -std=c99 */
  K&R: cc -Wall -Werror -O /* no -std flag — IS K&R by default */
      /* PCC (Portable C Compiler, 1978) or early GCC */
      /* warn_unused_result not available — use lint(1) */
      lint -u -h *.c /* era-correct static analysis */
